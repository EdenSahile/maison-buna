## Blocages / échecs

### Logo dans l'email — CID non rendu inline par Brevo

**Symptôme** : le logo apparaît en pièce jointe dans la boîte mail, pas dans le corps du mail.

**Approches tentées** :
1. Base64 dans `src="data:image/png;base64,..."` → bloqué par Gmail (sécurité)
2. URL `http://localhost:3000/images/logo-buna.png` → inaccessible depuis les serveurs Gmail
3. URL `raw.githubusercontent.com` → logo pas encore pushé sur GitHub
4. CID inline via `contentId` dans l'API Brevo → le logo s'envoie mais apparaît en PJ, pas inline

**Cause probable** : l'API REST Brevo (`POST /v3/smtp/email`) ne supporte pas le rendu inline CID de la même façon que le SMTP classique. Le paramètre `contentId` est peut-être ignoré ou traité différemment.

**Piste à explorer** : vérifier la doc Brevo pour les images inline — peut-être nécessite le champ `inlineImageActivation: true` ou une autre structure. Alternative : héberger le logo sur un domaine public (Cloudinary gratuit, ou déployer l'app sur Vercel/Railway avec `BASE_URL` en prod).
